## 1. 项目概述

罗宾出海分析 Agent 是一个微信小程序，面向已购买罗宾课程的学员提供**出海机会测评**服务。学员通过完成 18 道题（17 道选择题 + 1 道行业输入），获得一份基于 **「强成交人设」出海方法论** 的融合评估报告。AI 在答题过程中**后台静默逐题诊断**，最终在完整报告中一次性呈现深度个性化分析。报告先展示部分内容，学员留资后解锁完整报告，最终通过企业微信顾问承接 1 对 1 解读。

**业务支撑**：基于罗宾「强成交人设」轻资产出海方法论——定位定生死、内容定江山、SOP定天下。

**一句话定义**：一个为课程学员提供出海资质测评、AI 深度诊断并筛选高意向线索的获客工具。

## 2. 背景与痛点

### 2.1 业务背景
- 罗宾已有**稳定的学员群体**，学员多是有出海意向的国内商家
- 目前缺少一个**轻量筛选工具**来区分哪些学员真正具备出海条件、值得深度跟进
- 现有咨询模式依赖人工沟通，效率低，无法规模化

### 2.2 核心痛点
| 痛点 | 影响 | 产品解法 |
|------|------|---------|
| 学员不知道自己出海准备度如何 | 咨询门槛高，大量学员观望不行动 | 15 分钟测评给予量化反馈 |
| 人工无法规模化跟进所有学员 | 高潜学员被淹没 | 测评自动打分 + AI 报告 + 企微链路 |
| 学员没有动力留资 | 线索沉淀困难 | 部分报告先展示价值，留资后解锁完整报告 |

### 2.3 SWOT 分析
| 维度 | 分析 |
|------|------|
| **S 优势** | 已有学员基础，无需冷启动；测评形式轻量，用户完成率高 |
| **W 劣势** | 两人团队开发，排期紧张；微信审核时间不可控 |
| **O 机会** | 出海市场持续升温，商家测评需求增长；小程序生态成熟 |
| **T 威胁** | 算法备案可能要求关闭 AI 报告；竞品可能快速跟进同类工具 |

## 3. 产品目标与成功指标

### 3.1 产品目标
> **筛选有出海意向的学员，沉淀可跟进线索。**

1. **业务目标**: 6月26日前上线，累计沉淀 2000+ 有效留资线索
2. **用户目标**: 学员 3 分钟完成测评，5 分钟内获得出海评估价值
3. **体验目标**: 纯点击操作，零文字输入，零等待焦虑

### 3.2 成功指标 (KPI)
| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 测评完成率 | ≥ 70% | 完成测评 / 进入首页 UV |
| 留资转化率 | ≥ 30% | 留资数 / 完成测评数 |
| 测评耗时 | ≤ 3 分钟 | 从第一题到提交 |
| 报告生成成功率 | ≥ 99% | 成功生成 / 提交测评数 |
| 完整链路耗时 | ≤ 5 分钟 | 从进入小程序到看到完整报告 |

## 4. 目标用户与使用场景

### 4.1 用户画像

**核心用户：已购买罗宾课程的学员**

| 维度 | 描述 |
|------|------|
| 身份 | 已完成出海课程学习的商家创始人/高管 |
| 规模 | 从 1 人试水到 100 人以上团队不等 |
| 诉求 | 想知道自己学完课程后，实际出海条件如何 |
| 技术能力 | 普通微信用户，不要求技术背景 |
| 典型心理 | 对出海有兴趣但缺乏信心，需要外部验证 |

### 4.2 使用场景
1. **场景 A — 课后验证**: 学员刚学完课程，想立刻知道自己能打多少分
2. **场景 B — 定期复测**: 业务有变化后（如有了海外客户），重新测评看进步
3. **场景 C — 团队讨论**: 内部需要一份结构化的出海评估报告作为讨论材料
4. **场景 D — 咨询前准备**: 在联系企业微信前先完成测评，提高沟通效率

### 5.5 AI 逐题诊断机制（v2.0 新增）

**核心设计**：AI 不在前端打断用户，而是在后台逐题沉淀判断，最终汇总为深度报告。

```
用户答完一题 → 前端立即跳转下一题
              → 后端静默调用 DeepSeek API
              → 生成单题诊断记忆（report_memory）
              → 存入 ai_report_logs 或内存
              → 用户无感知

18 题全答完 → 汇总所有 report_memory
              → 结合总分、标签
              → 一次性生成融合版完整报告
```

**单题 AI 输出结构**（仅内部使用）：

```json
{
  "question_id": 6,
  "diagnosis_tag": ["获客依赖线下", "内容资产不足"],
  "report_memory": "该用户有海外获客基础，但线上内容获客能力不足，完整报告中应重点建议建立强成交人设内容矩阵。",
  "sales_hint": "后续顾问沟通可重点询问：目前海外客户主要来自哪些国家、展会线索转化周期多长。"
}
```

**最终报告结构**（融合版）：

```json
{
  "summary_report": {
    "total_score": 52,
    "tag": "基础具备型",
    "positioning_assessment": "定位评价（企业底座 + 出海现状）",
    "content_assessment": "内容力评价（产品力 + 内容力）",
    "conversion_assessment": "转化力评价（运营力 + 交付力）",
    "strengths": ["...", "..."],
    "risks": ["...", "..."],
    "unlock_hint": "提交信息解锁完整报告..."
  },
  "full_report": {
    "conclusion": "综合结论",
    "positioning_detail": "定生死 — 深度分析",
    "content_detail": "定江山 — 深度分析",
    "conversion_detail": "定天下 — 深度分析",
    "recommended_path": "推荐出海路径",
    "action_plan_30days": ["...", "..."],
    "consultant_guide": "顾问解读引导"
  }
}
```

## 7. 产品边界

## 8. 技术选型草案

## 11. 验收总标准

发版前必须满足以下条件：

- [ ] 全部 M 级功能测试通过
- [ ] 测评完成率 ≥ 70%（内测验证）
- [ ] AI 报告生成成功率 ≥ 99%（含兜底）
- [ ] 留资 -> 解锁完整报告 -> 展示企微二维码链路完整
- [ ] 无 AI 时模板报告可正常展示
- [ ] 所有接口强制 HTTPS
- [ ] 后端 API Key 未暴露在前端代码中
- [ ] 小程序审核材料（截图/录屏/隐私协议/AI 说明）已准备就绪
- [ ] 所有硬编码配置已提取到环境变量或数据库

## 12. 关键决策记录

| 决策 | 选项 | 结论 | 理由 |
|------|------|------|------|
| AI 模型 | DeepSeek / 国外模型 | DeepSeek | 国内模型审核更友好，成本更低 |
| 模板兜底 | 是否保留 | **保留** | AI 失败时保证用户体验 |
| 目标用户 | 公开流量 / 存量学员 | **存量学员** | 已有课程学员基础，转化效率更高 |
| 前端框架 | 原生 / Taro / UniApp | **原生** | 审核最快，两人团队减少框架复杂度 |
| 后台复杂度 | 完整后台 / 最小后台 | **最小后台** | 两人排期紧张，优先保证主链路 |
