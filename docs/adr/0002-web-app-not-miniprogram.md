# 0002-web-app-not-miniprogram

## 决策：放弃小程序，独立 Web 应用为唯一产品形态

**背景**：经评估，微信小程序存在三个无法逾越的限制：
1. 审核规则禁止聊天框/自由输入（Agent 核心交互模式）
2. `wx.cloud.callFunction` 不支持 SSE 流式返回（体验硬伤）
3. 封闭生态无 PC 端/SEO/链接分享（获客受限）

**决策**：不保留小程序分层，将全部功能迁移到独立 Web 应用（Python FastAPI 后端 + Web 前端）。

**原因**：
- 企业级 Agent 产品需要流式 AI 对话、PC 端访问、自由内容策略，这三个小程序都给不了
- 维护双平台（小程序 + Web）的 ROI 为负——团队只有两人（刘澳前端 + 你后端），双平台开发/测试/部署成本不可承受
- 旧链路 18 题问卷可以作为 Web 应用内的"快速测评"模式保留，不需要单独平台

**后果**：
- `cloudfunctions/` 和 `miniprogram/` 目录保留为历史参考，不再维护
- 微信生态的天然优势（OPENID 鉴权、企微联动、微信支付）全部丧失，需要替代方案
- Web 应用需要独立部署、独立鉴权、独立获客渠道
- 免 ICP 备案的 CloudBase 优势丧失——独立 Web 应用可能需要 ICP 备案
