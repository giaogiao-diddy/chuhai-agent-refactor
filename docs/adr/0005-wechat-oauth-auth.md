# 0005-wechat-oauth-auth

## 决策：微信扫码登录作为 Web 应用鉴权方式

**背景**：Web 应用无法使用小程序 OPENID 天然注入。目标用户为中国企业主，微信使用率接近 100%，扫码登录零摩擦。

**决策**：使用微信开放平台 OAuth 2.0（网站应用）实现扫码登录。

**原因**：
- 企业主不需要记密码、等验证码，扫码即登
- 可获取用户微信头像/昵称，个性化体验
- 与企微转化链路打通——微信登录用户添加企微顾问更自然
- 一个微信开放平台账号可同时绑定网站应用和企微应用

**后果**：
- 需要微信开放平台账号（不同于小程序账号），需企业资质认证（300 元/年）
- Python 后端需实现 OAuth 2.0 授权码流程
- 用户表从 CloudBase `users` 集合迁移到 MySQL，openid/unionid 为主键
- JWT Token 替代 CloudBase 会话管理
