## 1. 产品定位

**出海诊断 Agent** 是一款面向中国企业主的 AI 出海可行性诊断工具。用户通过自然对话完成企业信息采集（轮次不限，用户主导终止），AI 生成定制化出海诊断报告与 30 天行动计划，连接出海顾问完成 1v1 销售转化。

**核心价值**：AI 定制诊断报告 + 可执行行动计划 + 顾问 1v1 解读，三步闭环。

**与同类差异**：非固定问卷模板，而是 AI 驱动的流式对话体验——用户像跟真人顾问聊天一样完成诊断。

---

## 2. 目标用户

| 角色 | 描述 | 核心需求 | MVP |
|------|------|---------|:---:|
| 企业主 | 中小工厂老板（年营收 500 万-1 亿）、外贸公司/贸易商负责人 | "我适合出海吗？先做什么？要花多少钱？" | ✅ |
| 顾问 | 出海咨询公司的销售/咨询师 | 获取高质量线索、查看诊断报告、跟进成交 | ✅ |
| 管理员 | 咨询公司内部管理员 | 配置题目分值、查看团队数据、管理订阅 | ❌ 后续 |

**获客方式**：顾问在微信群/朋友圈分享链接 → 企业主扫码登录 → 对话诊断 → 留资 → 顾问跟进。

MVP 阶段为单租户（深度未来自己使用），多租户 SaaS 化后续迭代。

## 3. 核心用户旅程

### 3.1 企业主流程

```
1. 打开链接（顾问在微信群/朋友圈分享）
   ↓
2. 微信扫码登录 → 首次登录自动创建用户
   ↓
3. Agent 对话诊断（核心体验）
   - AI 开场白开启对话
   - 不限轮次自然对话，用户主导终止
   - SSE 流式回复（逐字显示，首 token < 1.5s）
   - AI 自动提取 7 维度企业信息（槽位填充）
   - Function Calling 查询市场商情
   - 向量 RAG 召回行业知识
   - 信息足够时提示"可以生成报告"，不强制终止
   - 2 次 AI 连续失败 → 降级标记 fallback_questionnaire
   ↓
4. 生成报告
   - 评分引擎算分打标签（纯规则）
   - 报告 Agent 生成诊断报告（DeepSeek）
   - 审计 Agent 校验（不合格带 feedback 重写，最多 3 次尝试）
   - 技术故障（DeepSeek 失败 / 审计 3 次不通过）→ 模板报告兜底
   - 信息不足 → 返回缺失项提示，不生成 0 分模板
   ↓
5. 查看部分报告（留资前可见）
   展示：
   - total_score（可行性总分）
   - display_score（展示分）
   - tag（标签：观察准备型 / 轻量试探型 / 基础具备型 / 优先布局型）
   - tag_explanation（标签解读，≤80字）
   - preliminary_judgment（综合诊断，≤120字）
   - strengths（优势列表，2-3条）
   - risks（风险列表，2-3条）
   - unlock_hint（引导留资文案："添加企业微信顾问解锁完整报告和1v1深度解读"）
   ↓
6. 留资解锁 → 填写手机号（无需验证码）→ 解锁完整报告
   新增展示：
   - summary_conclusion（综合结论，≤400字）
   - positioning_assessment（定位分析，≤300字）
   - content_assessment（内容/产品适配分析，≤300字）
   - conversion_assessment（转化路径分析，≤300字）
   - dimension_scores（7 维度雷达图得分明细）
   - recommended_path（推荐出海路径，≤250字）
   - risk_reminder（风险提醒，≤250字）
   - action_plan_30days（30 天行动计划，4 步）
   - consultant_guide（顾问引导语，引导添加企微）
   展示：企微二维码，引导添加顾问
   ↓
7. "我的报告"页面
   可随时回看历史诊断、追踪进展
   顾问更新跟进状态后用户可见
```

### 3.2 顾问流程

```
1. 微信扫码登录管理后台（与企业主使用同一 OAuth，系统识别角色后跳转顾问后台）
2. 线索列表（按 P0→P3 优先级排序）
   显示：企业名、行业、标签、总分、时间
3. 点击线索 → 查看详情：
   - 企业基本盘
   - 顾问版报告（含 lead_score、跟进优先级、销售话术）
   - 跟进状态（未联系 / 已联系 / 已预约 / 已成交）
4. 更新跟进状态 + 备注
```

## 4. 功能模块

### 4.1 企业主端（4 个模块）

| 模块 | 功能点 |
|------|--------|
| **微信登录** | 微信开放平台 OAuth 2.0 扫码登录、自动注册、JWT 会话保持 |
| **Agent 对话诊断** | SSE 流式对话、AI 槽位提取（7 维度）、Function Calling 市场商情查询、向量 RAG 行业知识检索、轮次不限用户主导终止、2 次失败降级 |
| **诊断报告** | 部分报告（留资前可看摘要）+ 完整报告（留资后解锁全部维度）、用户版/顾问版物理隔离、AI 报告 / 模板兜底双轨、历史报告回看 |
| **留资** | 手机号填写（无需验证码）→ 解锁完整报告、企微二维码展示 |

### 4.2 顾问端（2 个模块）

| 模块 | 功能点 |
|------|--------|
| **线索列表** | 按 P0/P1/P2/P3 优先级排序、显示总得分/标签/行业/时间 |
| **线索详情** | 用户基本盘、顾问版报告（含 lead_score + 销售话术 + 跟进重点）、跟进状态更新 + 备注 |

### 4.3 本期不做（后续迭代）

- 管理后台（题目配置、分值修改、团队管理）
- 多租户（MVP 深度未来单租户）
- 企微机器人自动推送
- 无出海经验分支（等老板提供题目后再开发）
- 顾问端注册/权限系统（MVP 在数据库 users 表手动标记 role=consultant，登录方式与企业主相同：微信扫码）

## 5. 非功能需求

| 维度 | 要求 | 理由 |
|------|------|------|
| 对话延迟 | 首 token < 1.5s，流式输出 | SSE 真流式，用户感知不可回头 |
| 报告生成 | < 30s（含 2 次审计重试） | DeepSeek 60s timeout，双保险保证 100% 出报告 |
| 并发 | MVP 20 人同时对话 | 首批客户规模 |
| 数据安全 | 用户版报告不含 lead_score / sales_followup | 代码级物理隔离 |
| AI 容错 | AI 失败自动降级，用户无感知；信息不足返回提示而非模板 | 2 次连续 AI 失败 → fallback_questionnaire；模板兜底仅用于技术故障 |
| Token 成本 | 单次诊断 ≤ 15,000 tokens | ≈ ¥0.03-0.05/次 |
| 数据库 | PostgreSQL ACID 事务，reports + lead_reports 原子写入 | 两份报告同生共死 |
| 浏览器 | Chrome / Edge / Safari 最新两版 | PC 端为主 |

---

## 6. 技术约束

- **后端**：Python 3.12 / FastAPI / LangGraph / pgvector
- **前端**：React 19 / Next.js (App Router) / shadcn/ui
- **数据库**：PostgreSQL 16+ + pgvector（一套数据库同时满足 ACID 事务 + 向量检索）
- **AI**：DeepSeek API（流式对话、报告生成、text-embedding）
- **部署**：CloudBase Run (Docker)
- **鉴权**：微信开放平台 OAuth 2.0 + JWT

## 7. 成功指标（MVP）

| 指标 | 目标值 |
|------|--------|
| 对话完成率 | ≥ 70% 用户完成诊断并生成报告 |
| 报告生成率 | 100%（含模板兜底） |
| 留资转化率 | ≥ 30% 用户填写手机号 |
| 首 token 延迟 | < 1.5s（P95） |
| 单次诊断平均耗时 | < 5 分钟（含对话 + 报告生成） |
| 系统可用性 | 顾问后台可正常查看线索 |
