# 企业微信自动解锁卡点说明

## 一、结论

当前小程序想实现“用户添加企业微信客服后，系统自动解锁完整报告”，核心卡点不在代码，而在账号主体和企业微信权限。

目前遇到的问题是：

```text
小程序开发主体与企业微信主体不一致
```

在这种情况下，企业微信客户联系能力受限，无法稳定使用客户联系 Secret、带参数联系我二维码和客户添加回调。

因此，当前条件下无法实现严格意义上的：

```text
用户真实添加客服后，系统自动识别并精准解锁对应报告
```

## 二、自动解锁真正需要什么

要实现可靠自动解锁，系统必须同时做到三件事：

```text
1. 知道用户是哪一个小程序用户
2. 知道用户添加了哪一个企业微信客服
3. 知道这次添加行为对应哪一份测评报告
```

小程序本身可以识别用户，但企业微信添加客服这个动作发生在企业微信体系内。

所以必须依赖企业微信客户联系能力，把“添加客服事件”回传给系统。

可靠方案需要：

```text
企业微信客户联系 Secret
客户联系 API
带 state 参数的联系我二维码
客户添加事件回调
```

其中 `state` 用来绑定：

```text
用户测评报告 ID
```

用户添加客服后，企业微信回调把 `state` 返回，系统才能自动找到对应报告并解锁。

## 三、当前为什么卡住

当前在获取客户联系 Secret 时，企业微信提示主体不一致。

这意味着：

```text
小程序主体和企业微信主体不是同一个企业/组织
```

在主体不一致的情况下，企业微信客户联系 API 能力无法正常打通。

直接影响是：

```text
不能稳定生成带参数的企业微信联系我二维码
不能接收有效的客户添加回调
不能用企业微信事件精准判断用户是否添加客服
不能自动把添加客服用户和测评报告对应起来
```
